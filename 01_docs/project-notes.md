# RO FIKA 改修メモ

## 2026-07-14: 光のセットを再設計

- 課題: 単品写真3枚を均等に並べたため、980円の「光のセット」が一つの商品として読めなかった。モバイルでは縦リストになり、冷たい飲みものと焼き菓子の関係も見えなかった。
- 決定: 窓辺でドリンクと焼き菓子が並ぶ専用写真を主役にし、価格・セット内容・選べる品目・季節写真への導線を一つのブロックに集約する。
- デザイン: 既存の生成り、深緑、窓辺の光を維持。均等な3カードを避け、主写真と補助サムネイルで強弱を作る。
- 目標: 初見で「何がセットか」「いくらか」「何を選べるか」を一画面内で理解できること。

## 2026-07-14: 画像とメニューの紐付け

- 決定: 補助サムネイルは装飾ではなく、対応するメニュー説明へ移動するリンクとして扱う。加えて、対象商品のメニュー行そのものにも同じ画像を表示する。
- 挙動: 選んだ画像のメニュー行をハッシュ遷移で表示し、背景と左線で到達先を示す。各行では画像・商品名・説明を一体で読める。

## 2026-07-14: 全品のメニュー写真を統一

- 決定: 写真の有無で商品に優劣があるように見せないため、掲載する6品すべてに商品写真を付ける。
- 追加: 水出しのお茶、ブルーベリークランブル、バタースコーン。既存の3品と同じ窓辺の自然光、アイボリーリネン、淡い陶器、深緑の影で統一する。
- 表示: PCは小〜中サイズの縦写真と商品名・説明を1行に、モバイルは写真の下に商品名・説明を置く。主役の「光のセット」だけは従来どおり完成形を大きく見せる。

## 2026-07-14: モバイルの光のセットを案内へ整理

- 課題: スマホで補助写真を横スクロールさせると、画面外の品目に気づきにくく、セット枠が注文・選択画面のように見えた。通常メニューにも全6品の写真があり、同じ情報を重複していた。
- 決定: セット枠の役割を「完成形とセット内容を伝える案内」に限定。横スクロールと、商品カードをタップして選択する挙動は取り除く。
- デザイン: `Drink` と `Bake` の2枚をわずかに重ね、飲みもの1杯と焼き菓子1品が一つのセットになることだけを視覚化する。カードは操作部品ではない。
- 導線: 「対象メニューを見る」から冷たい飲みものの見出しへ移動し、全6品の写真・商品名・説明は通常メニュー側で読む。アンカー位置には固定ヘッダー分の余白を確保する。
